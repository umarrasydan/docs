### Error Handling Template

All providers share common error handling patterns. Use this template for robust error management:

<CodeGroup>
```kotlin Android Error Handling
fun handleTerminalError(error: TerminalError?) {
    when (error?.code) {
        ErrorCode.AUTHENTICATION_FAILED -> {
            // Verify API key and terminal ID
            Log.e("Terminal", "Authentication failed: ${error.message}")
            showAuthenticationErrorDialog()
        }
        
        ErrorCode.TERMINAL_BUSY -> {
            // Terminal is processing another transaction
            Log.w("Terminal", "Terminal busy: ${error.message}")
            retryAfterDelay(5000) // 5 seconds
        }
        
        ErrorCode.FAILED_TO_CONNECT, ErrorCode.NOT_CONNECTED -> {
            // Network connectivity issues
            Log.e("Terminal", "Connection failed: ${error.message}")
            checkNetworkConnectivity()
        }
        
        ErrorCode.KEY_INVALID, ErrorCode.INVALID_CREDENTIAL -> {
            // Credential issues
            Log.e("Terminal", "Invalid credentials: ${error.message}")
            refreshCredentials()
        }
        
        else -> {
            // Log and escalate unknown errors
            Log.e("Terminal", "Unknown error: ${error?.message}")
            reportError(error)
        }
    }
}
```

```swift iOS Error Handling
func handleTerminalError(_ error: TerminalError?) {
    guard let error = error else { return }
    
    switch error.code {
    case ErrorCode.authenticationFailed:
        // Verify API key and terminal ID
        print("Authentication failed: \(error.message)")
        showAuthenticationErrorDialog()
        
    case ErrorCode.terminalBusy:
        // Terminal is processing another transaction
        print("Terminal busy: \(error.message)")
        DispatchQueue.main.asyncAfter(deadline: .now() + 5) {
            // Retry after 5 seconds
            retryLastOperation()
        }
        
    case ErrorCode.failedToConnect, ErrorCode.notConnected:
        // Network connectivity issues
        print("Connection failed: \(error.message)")
        checkNetworkConnectivity()
        
    case ErrorCode.keyInvalid, ErrorCode.invalidCredential:
        // Credential issues
        print("Invalid credentials: \(error.message)")
        refreshCredentials()
        
    default:
        // Log and escalate unknown errors
        print("Unknown error: \(error.message)")
        reportError(error)
    }
}
```

## SDK Types Explained

<Tabs>
<Tab title="Terminal H2H (Host-to-Host)">
  **Recommended for most integrations**
  
  - Direct connection between your application and physical terminals
  - Centralized fleet management through gateway application  
  - Better for production environments with multiple terminals
  - Requires Terminal Gateway app installation on the device
  
  **Best for**: Retail environments, multi-terminal setups, enterprise deployments
</Tab>

<Tab title="Terminal C2C (Client-to-Client)">
  **For direct terminal communication**
  
  - Direct REST API communication with terminals over local network
  - No gateway app required for some providers (like Cashup)
  - Simpler setup for single-terminal scenarios
  - Uses separate OpenAPI specification
  
  **Best for**: Single-terminal setups, embedded systems, custom integrations
</Tab>
</Tabs>

## Platform-Specific Notes

### Android (Kotlin)

- **Minimum SDK**: Android 5.0 (API level 21)
- **Dependencies**: Use Maven Local repository included in SDK package
- **Gradle Configuration**: Both Gradle Groovy and Kotlin DSL supported
- **Provider Dependencies**: Optional, include only what you need

### iOS (Swift)

- **Requirements**: iOS 12.0+ 
- **Dependencies**: Sentry cocoapod required
- **Installation**: Add TerminalGateway.xcframework to your project
- **Provider Methods**: Available without additional dependencies

<Accordion title="Environment-Specific Configuration">
Configure different settings for development vs production:

```kotlin
class MyApp : Application() {
    override fun onCreate() {
        super.onCreate()

        val mode = if (BuildConfig.DEBUG) {
            TerminalMode.INTEGRATION
        } else {
            TerminalMode.LIVE
        }

        TerminalApp.initialize(
            application = this,
            clientKey = if (BuildConfig.DEBUG) TEST_CLIENT_KEY else LIVE_CLIENT_KEY,
            mode = mode
        )
    }
}
```
</Accordion>

<Accordion title="Error Handling Setup">
Implement comprehensive error handling from the start:

```kotlin
TerminalH2H.observeError(
    CoroutineScope(Dispatchers.IO)

    when (error?.code) {
        ErrorCode.AUTHENTICATION_FAILED -> {
            // Handle auth errors
            showAuthenticationErrorDialog()
        }

            // Handle busy terminal
```
