## Overview

Use Xendit's Terminal API with Terminal Gateway to process in-person payments on physical terminals. You create and manage payment sessions via the Terminal API and connect to devices using Terminal Gateway (desktop app or mobile SDKs).

<Info>
You need a Terminal API key (different from standard Dashboard keys) and at least one configured terminal device connected via Terminal Gateway.
</Info>

## Prerequisites

Before you begin, ensure you have:

- Terminal API access and API key from the Xendit In-Person Payment team
- Terminal Gateway set up (desktop app or Android/iOS SDK)
- A terminal with Terminal ID and network connectivity
- Your environment base URL:
  - Development: `https://terminal-dev.xendit.co`
  - Production: `https://terminal.xendit.co`

<Warning>
Terminal API uses HTTP Basic Auth with your Terminal API key as the username and an empty password. Always include a trailing colon when constructing the header.
</Warning>

## How it works

- **Create a session** with Terminal API for an amount, currency, and terminal.
- **Execute on device** through Terminal Gateway; the customer taps/inserts/swipes.
- **Receive callbacks** for real-time status updates.
- **Query details** for reconciliation or show receipts.

## Start here

<Steps>
<Step title="Obtain Terminal API key">
  Contact the Xendit In-Person Payment team to request your Terminal API key.
  
  <Check>
  You can successfully authenticate to the development base URL with your key.
  </Check>
</Step>

<Step title="Set up Terminal Gateway">
  Install and connect the Terminal Gateway desktop app or integrate the mobile SDK on your device. Pair your terminal and confirm it shows a connected/ready status.
  
  <CardGroup cols={2}>
    <Card title="Gateway App Configuration" icon="settings" href="/api-reference/terminal-gateway/app-configuration" />
    <Card title="Android SDK" icon="android" href="/api-reference/terminal-gateway/android-sdk" />
    <Card title="iOS SDK" icon="apple" href="/api-reference/terminal-gateway/ios-sdk" />
  </CardGroup>
  
  <Check>
  The device appears connected and ready in your Gateway.
  </Check>
</Step>

<Step title="Create your first payment session">
  Use the Terminal API to create a `PAY` session for your terminal.
  
  <RequestExample>
  ```bash cURL
  curl -X POST 'https://terminal-dev.xendit.co/v1/terminal/sessions' \
    -u 'YOUR_TERMINAL_API_KEY:' \
    -H 'Content-Type: application/json' \
    -H 'Idempotency-key: unique-key-123' \
    -d '{
      "session_type": "PAY",
      "mode": "TERMINAL",
      "currency": "IDR",
      "amount": 10000,
      "country": "ID",
      "channel_properties": { "terminal_id": "TERM001" }
    }'
  ```
  </RequestExample>

  <Check>
  You receive a session ID and the terminal prompts for payment.
  </Check>
</Step>

<Step title="Handle callbacks and confirm">
  Configure your webhook endpoint to receive payment status updates, then fetch payment details to confirm final status.
  
  <CardGroup cols={2}>
    <Card title="Callbacks" icon="webhook" href="/api-reference/terminal-api/callbacks" />
    <Card title="Get Payment Detail" icon="receipt" href="/api-reference/terminal-api/get-payment-detail" />
  </CardGroup>
  
  <Tip>
  Use unique `Idempotency-key` values per create request to safely retry.
  </Tip>
</Step>
</Steps>
