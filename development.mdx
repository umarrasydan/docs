<Info>
  **Prerequisites**:
  - Node.js version 16 or higher
  - A Xendit account with API credentials
  - Your preferred programming language environment
</Info>

Follow these steps to set up your development environment for Xendit integration.

<Steps>
<Step title="Install the Xendit SDK">

```bash
npm install xendit-node
```

Or for other languages:
```bash
# Python
pip install xendit

# PHP
composer require xendit/xendit-php

# Java
# Add to your pom.xml or build.gradle
</Step>

<Step title="Set up environment variables">

Create a `.env` file in your project root:

```bash
XENDIT_SECRET_KEY=your_secret_key_here
XENDIT_PUBLIC_KEY=your_public_key_here
```

<Warning>Never commit your secret keys to version control!</Warning>

</Step>
</Steps>

## Testing your integration

Use Xendit's sandbox environment to test your integration before going live:

```javascript
// Example: Create a virtual account
const xendit = require('xendit-node');
const client = new xendit.XenditClient({
  secretKey: process.env.XENDIT_SECRET_KEY,
});

const virtualAccount = await client.VirtualAccount.create({
  externalId: 'va-external-id',
  bankCode: 'BCA',
  name: 'John Doe',
  virtualAccountNumber: '1234567890',
});
```

## Webhook testing

Test your webhook endpoints using tools like ngrok or similar:

```bash
# Install ngrok
npm install -g ngrok

# Expose your local server
ngrok http 3000
```

Use the ngrok URL as your webhook URL in the Xendit dashboard.

## Going live

When you're ready to go live:

1. Switch from sandbox to live API keys
2. Update your webhook URLs to production endpoints
3. Test thoroughly with small amounts first
4. Monitor your integration for any issues

## Code formatting

We recommend using proper code formatting tools for your chosen language:

- **JavaScript/Node.js**: ESLint + Prettier
- **Python**: Black + flake8
- **PHP**: PHP CS Fixer
- **Java**: Google Java Format

## Troubleshooting

<AccordionGroup>
  <Accordion title='Error: Invalid API key'>

    Make sure you're using the correct API key for your environment:
    - Use sandbox keys for testing
    - Use live keys for production
    - Check that the key is properly set in your environment variables
  </Accordion>

  <Accordion title="Issue: Webhook not receiving events">
  
    Common solutions:
    1. Ensure your webhook URL is publicly accessible
    2. Check that your server is running and responding
    3. Verify the webhook URL in your Xendit dashboard
    4. Check your server logs for any errors
  </Accordion>
</AccordionGroup>
