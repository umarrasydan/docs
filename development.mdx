## Prerequisites

Before setting up your development environment, ensure you have:

- **Node.js 16+** or your preferred programming language environment
- **Xendit account** with API credentials ([Sign up here](https://dashboard.xendit.co))
- **Text editor or IDE** with syntax highlighting for your language
- **Git** for version control (recommended)

<Info>
**New to development?** This guide assumes basic familiarity with programming concepts. For complete beginners, start with our [quickstart guide](/quickstart).
</Info>

## Environment Setup

Follow these steps to configure your development environment:

<Steps>
<Step title="Install the Xendit SDK">
  Install the Xendit SDK for your preferred programming language:
  
  <Tabs>
  <Tab title="Node.js">
    ```bash
    npm install xendit-node
    ```
    
    <Check>
    Verify installation by running `npm list xendit-node` in your project directory.
    </Check>
  </Tab>
  
  <Tab title="Python">
    ```bash
    pip install xendit
    ```
    
    <Check>
    Verify installation by running `python -c "import xendit; print(xendit.__version__)"`.
    </Check>
  </Tab>
  
  <Tab title="PHP">
    ```bash
    composer require xendit/xendit-php
    ```
    
    <Check>
    Verify installation by checking your `composer.json` file for the xendit package.
    </Check>
  </Tab>
  
  <Tab title="Java">
    Add to your `pom.xml`:
    ```xml
    <dependency>
      <groupId>com.xendit</groupId>
      <artifactId>xendit-java</artifactId>
      <version>3.0.0</version>
    </dependency>
    ```
    
    <Check>
    Verify installation by running `mvn dependency:tree | grep xendit`.
    </Check>
  </Tab>
  </Tabs>
</Step>

<Step title="Configure environment variables">
  Set up secure environment variable management:
  
  Create a `.env` file in your project root:
  
  ```bash .env
  # Xendit API Configuration
  XENDIT_SECRET_KEY=your_secret_key_here
  XENDIT_PUBLIC_KEY=your_public_key_here
  
  # Environment
  NODE_ENV=development
  
  # Optional: Webhook configuration
  WEBHOOK_URL=https://your-domain.com/webhook
  ```
  
  <Warning>
  **Security**: Never commit your `.env` file to version control. Add it to your `.gitignore` file.
  </Warning>
  
  <Tip>
  Use different API keys for development and production environments to maintain security.
  </Tip>
</Step>

<Step title="Initialize your project">
  Create a basic project structure and initialize the Xendit client:
  
  ```javascript index.js
  const xendit = require('xendit-node');
  require('dotenv').config();
  
  // Initialize Xendit client
  const client = new xendit.XenditClient({
    secretKey: process.env.XENDIT_SECRET_KEY,
  });
  
  // Test connection
  async function testConnection() {
    try {
      const response = await client.VirtualAccount.getBanks();
      console.log('‚úÖ Xendit client initialized successfully');
      console.log(`Available banks: ${response.length}`);
    } catch (error) {
      console.error('‚ùå Failed to initialize Xendit client:', error.message);
    }
  }
  
  testConnection();
  ```
  
  <Check>
  Run your initialization script and verify you see "Xendit client initialized successfully".
  </Check>
</Step>
</Steps>

## Testing Your Integration

Use Xendit's sandbox environment to safely test your integration before going live:

<Steps>
<Step title="Set up sandbox testing">
  Configure your application to use sandbox API keys for testing:
  
  ```javascript test-config.js
  const xendit = require('xendit-node');
  require('dotenv').config();
  
  // Use sandbox keys for testing
  const client = new xendit.XenditClient({
    secretKey: process.env.XENDIT_SECRET_KEY, // Sandbox key
  });
  
  // Test virtual account creation
  async function testVirtualAccount() {
    try {
      const virtualAccount = await client.VirtualAccount.create({
        externalId: 'test-va-' + Date.now(),
        bankCode: 'BCA',
        name: 'Test Customer',
        virtualAccountNumber: '1234567890',
        suggestedAmount: 10000,
        isClosed: true,
        expectedAmount: 10000,
        expirationDate: new Date(Date.now() + 24 * 60 * 60 * 1000),
        isSingleUse: true,
      });
      
      console.log('‚úÖ Virtual Account created:', virtualAccount.accountNumber);
      return virtualAccount;
    } catch (error) {
      console.error('‚ùå Virtual Account creation failed:', error.message);
    }
  }
  
  testVirtualAccount();
  ```
  
  <Check>
  Run your test script and verify you can create virtual accounts successfully.
  </Check>
</Step>

<Step title="Test webhook endpoints">
  Set up webhook testing using ngrok or similar tools:
  
  ```bash
  # Install ngrok globally
  npm install -g ngrok
  
  # Expose your local server
  ngrok http 3000
  ```
  
  Use the ngrok URL as your webhook URL in the Xendit dashboard:
  
  ```javascript webhook-test.js
  const express = require('express');
  const app = express();
  
  app.use(express.json());
  
  // Webhook endpoint for testing
  app.post('/webhook', (req, res) => {
    console.log('üì® Webhook received:', JSON.stringify(req.body, null, 2));
    
    // Verify webhook signature (recommended)
    const signature = req.headers['x-xendit-signature'];
    
    // Process payment notification
    const { status, external_id, amount } = req.body;
    
    if (status === 'PAID') {
      console.log('‚úÖ Payment completed!');
      console.log(`External ID: ${external_id}`);
      console.log(`Amount: ${amount}`);
    }
    
    res.status(200).send('OK');
  });
  
  app.listen(3000, () => {
    console.log('üöÄ Webhook server running on port 3000');
    console.log('üì° Use ngrok to expose this server for webhook testing');
  });
  ```
  
  <Tip>
  Use webhook testing tools like [webhook.site](https://webhook.site) to verify your endpoint works before configuring in the dashboard.
  </Tip>
</Step>
</Steps>

## Production Deployment

When you're ready to deploy to production, follow these steps:

<Steps>
<Step title="Switch to live API keys">
  Update your environment variables to use live API keys:
  
  ```bash .env.production
  # Production API keys
  XENDIT_SECRET_KEY=your_live_secret_key
  XENDIT_PUBLIC_KEY=your_live_public_key
  
  # Environment
  NODE_ENV=production
  
  # Production webhook URL
  WEBHOOK_URL=https://your-domain.com/webhook
  ```
  
  <Warning>
  **Critical**: Test thoroughly with small amounts before processing real payments.
  </Warning>
</Step>

<Step title="Update webhook URLs">
  Configure production webhook endpoints in your Xendit dashboard:
  
  - Update webhook URLs to your production domain
  - Ensure your production server is accessible from the internet
  - Test webhook delivery with a test payment
  
  <Check>
  Verify webhook notifications are received correctly in production.
  </Check>
</Step>

<Step title="Implement monitoring">
  Set up monitoring and logging for your payment integration:
  
  ```javascript monitoring.js
  const winston = require('winston');
  
  // Configure logging
  const logger = winston.createLogger({
    level: 'info',
    format: winston.format.combine(
      winston.format.timestamp(),
      winston.format.json()
    ),
    transports: [
      new winston.transports.File({ filename: 'error.log', level: 'error' }),
      new winston.transports.File({ filename: 'combined.log' }),
    ],
  });
  
  // Log payment events
  function logPaymentEvent(event, data) {
    logger.info('Payment Event', {
      event,
      timestamp: new Date().toISOString(),
      data: {
        external_id: data.external_id,
        amount: data.amount,
        status: data.status,
      },
    });
  }
  ```
  
  <Tip>
  Set up alerts for failed payments and webhook delivery failures to ensure reliable operation.
  </Tip>
</Step>
</Steps>

## Troubleshooting

<AccordionGroup>
<Accordion title="Invalid API Key Error">
  **Problem**: Getting "Invalid API key" errors when making API calls
  
  **Solutions**:
  - Verify you're using the correct environment (sandbox vs live)
  - Check that your secret key is properly set in environment variables
  - Ensure there are no extra spaces or characters in your API key
  - Confirm the API key has the correct permissions for your use case
  
  <Check>
  Test your API key by making a simple API call to verify authentication.
  </Check>
</Accordion>

<Accordion title="Webhook Not Receiving Events">
  **Problem**: Webhooks not being delivered to your endpoint
  
  **Solutions**:
  - Ensure your webhook URL is publicly accessible (use ngrok for local testing)
  - Check that your server responds with HTTP 200 status
  - Verify the webhook URL in your Xendit dashboard
  - Check server logs for any errors or timeouts
  - Test webhook delivery with webhook testing tools
  
  <Tip>
  Use [webhook.site](https://webhook.site) to test your webhook endpoint before configuring it in the dashboard.
  </Tip>
</Accordion>

<Accordion title="Payment Creation Failures">
  **Problem**: Unable to create payments or virtual accounts
  
  **Solutions**:
  - Verify all required parameters are provided
  - Check that bank codes and payment methods are valid
  - Ensure amounts are within acceptable ranges
  - Review error messages for specific validation issues
  - Test with sandbox environment first
  
  <Warning>
  Always test with sandbox environment before attempting live payments.
  </Warning>
</Accordion>

<Accordion title="Environment Configuration Issues">
  **Problem**: Environment variables not loading correctly
  
  **Solutions**:
  - Verify `.env` file is in the correct location
  - Check that you're loading environment variables in your application
  - Ensure `.env` file is not committed to version control
  - Use different configuration files for different environments
  
  <Check>
  Add logging to verify environment variables are loaded correctly.
  </Check>
</Accordion>
</AccordionGroup>
