# Get Public Payment Detail

Retrieves the details of a specific payment transaction using its unique identifier. This is a public endpoint that doesn't require authentication.

<Note>
This endpoint is public and doesn't require authentication. It's typically used for retrieving payment details in scenarios where you don't have access to the authenticated API.
</Note>

## Headers

<ParamField query="Content-Type" type="string" required>
  `application/json`
</ParamField>

## Path Parameters

<ParamField path="id" type="string" required>
  The unique identifier of the payment to retrieve
  
  Must match the pattern: `^[0-9a-fA-F]{8}\-[0-9a-fA-F]{4}\-[0-9a-fA-F]{4}\-[0-9a-fA-F]{4}\-[0-9a-fA-F]{12}$`
</ParamField>

## Response

<ResponseField name="payment_id" type="string">
  Xendit unique Payment ID
</ResponseField>

<ResponseField name="business_id" type="string">
  Xendit business ID
</ResponseField>

<ResponseField name="reference_id" type="string">
  Merchant's reference ID

<ResponseField name="customer_id" type="string">
  Xendit-generated Customer ID
</ResponseField>

  <Enum values={["PAY", "PRE_AUTH"]} />

</ResponseField>

<ResponseField name="country" type="string">
  <Enum values={["ID", "VN", "TH"]} />
  
  Country code
</ResponseField>

  <Enum values={["IDR", "VND", "THB"]} />

</ResponseField>

<ResponseField name="payment_details.payment_method" type="string">
  <Enum values={["ID_INSERT_CARD", "ID_CONTACTLESS", "ID_BRIZZI", "ID_QRIS", "VN_CARD", "VN_QR", "TH_ALIPAY_PLUS", "TH_WECHATPAY", "TH_TRUEMONEY", "TH_LINEPAY", "TH_SHOPEEPAY", "TH_SABUYMONEY", "TH_MAXME", "TH_CARD", "TH_BBL_QRCS", "TH_MULTIIPP", "TH_QRCODE", "MY_QR", "MY_CARD", "MY_EWALLET"]} />

  Payment failure code (nullable)
</ResponseField>

  Key-value entries for your custom data

  Payment's creation time (ISO 8601 format)
</ResponseField>

<ResponseField name="updated" type="string">
  Payment's updated time (ISO 8601 format)
</ResponseField>

## Example Request

<CodeGroup>
```bash cURL
curl -X GET https://terminal-dev.xendit.co/public/v1/terminal/payments/pay_1234567890abcdef \
  -H "Content-Type: application/json"
```

```javascript JavaScript
const response = await fetch('https://terminal-dev.xendit.co/public/v1/terminal/payments/pay_1234567890abcdef', {
  method: 'GET',
  headers: {
    'Content-Type': 'application/json'
  }
});
```

```python Python
import requests

response = requests.get(
    'https://terminal-dev.xendit.co/public/v1/terminal/payments/pay_1234567890abcdef',
    headers={
        'Content-Type': 'application/json'
    }
)
```
</CodeGroup>

## Payment Status Values

<CardGroup cols={3}>
  <Card title="SUCCEEDED" icon="check-circle" color="green">
    Payment was successfully processed and completed
  </Card>
  <Card title="VOIDED" icon="x-circle" color="red">
    Payment was voided before settlement
  </Card>
  <Card title="AUTHORIZED" icon="clock" color="blue">
    Payment was authorized but not yet captured (Pre-Auth only)
  </Card>
</CardGroup>

## Payment Types

<CardGroup cols={2}>
  <Card title="PAY" icon="credit-card">
    One-time payment that is immediately captured
  </Card>
  <Card title="PRE_AUTH" icon="lock">
    Pre-authorization that holds funds but doesn't capture them immediately
  </Card>
</CardGroup>
