# Terminal API

This documentation provides details on the API to facilitate in-person payment using terminal devices. It covers how to create payment sessions, retrieve session details, void payment requests, push operation commands to terminals, and handle terminal callbacks.

## Key Features

- Create new payment sessions and retrieve session details
- Push operational commands to terminal devices
- Receive in-person payment callbacks for payment events
- Handle errors and manage different payment scenarios

## Getting Started

You'll need an In-Person Payment API_KEY from the Xendit In-Person Payment team. This will be a different key from the ones you obtain from the Xendit Dashboard. We assume you have registered for the Xendit Dashboard.

### Terminal Gateway App/SDK

This application or SDK is mandatory. It streamlines the management of your terminal devices.

#### Initializing the Terminal Gateway App/SDK

- **For app configuration**: Please refer to this doc for more information. Windows and Mac OS are available
- **For SDK**: Please refer to the respective doc for more information. Android (Kotlin) and iOS (Swift) are available

## Authentication

Merchants authenticate with the Xendit API using basic authentication.

- **Username**: API_KEY
- **Password**: (Empty string)

<Warning>
Make sure to include `:` at the end of the API key when encoding for basic authentication.
</Warning>
