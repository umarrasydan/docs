# Void Payment

Voids a completed payment transaction before it settles. This is useful for canceling a transaction that has been processed but not yet settled.

<Warning>
Voiding is only possible before a payment settles. After settlement, you would need to process a refund instead.
</Warning>

## Headers

<ParamField query="Content-Type" type="string" required>
  `application/json`
</ParamField>

<ParamField query="Authorization" type="string" required>
  `Basic <Base64Encode(API_KEY:)>`
</ParamField>

## Path Parameters

<ParamField path="payment_id" type="string" required>
  The unique identifier of the payment to void
</ParamField>

## Response

<ResponseField name="code" type="string">
  Response Code
  
  SUCCESS if the request is successful
</ResponseField>

<ResponseField name="error_code" type="string">
  Response Error Code
</ResponseField>

<ResponseField name="message" type="string">
  Response Message
</ResponseField>

## Example Request

<CodeGroup>
```bash cURL
curl -X POST https://terminal-dev.xendit.co/v1/terminal/payments/pay_1234567890abcdef/void \
  -H "Content-Type: application/json" \
  -H "Authorization: Basic <Base64Encode(API_KEY:)>"
```

```javascript JavaScript
const response = await fetch('https://terminal-dev.xendit.co/v1/terminal/payments/pay_1234567890abcdef/void', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json',
    'Authorization': 'Basic ' + btoa('API_KEY:')
  }
});
```

```python Python
import requests
import base64

api_key = "your_api_key"
auth_string = base64.b64encode(f"{api_key}:".encode()).decode()

response = requests.post(
    'https://terminal-dev.xendit.co/v1/terminal/payments/pay_1234567890abcdef/void',
    headers={
        'Content-Type': 'application/json',
        'Authorization': f'Basic {auth_string}'
    }
)
```
</CodeGroup>
