# Terminal Client

Xendit's Terminal Client SDK enables you to integrate secure payment processing directly into your applications. Connect to physical payment terminals and process transactions seamlessly across multiple platforms with our comprehensive Kotlin Multiplatform (KMP) compatible SDK.

## What you can do with Terminal Client

Terminal Client provides everything you need to build in-person payment experiences:

- **Process payments** directly through terminal devices using a unified API
- **Manage terminal connections** and handle device configuration
- **Handle transactions** including payments, cancellations, receipts, and settlements
- **Support multiple platforms** including Android, iOS, and other KMP-compatible platforms
- **Connect to local gateway** for secure communication with payment terminals

<Info>
Terminal Client works alongside the Terminal Gateway to provide a complete in-person payment solution. You'll use Terminal Client to interact with the local gateway service that communicates with physical payment devices.
</Info>

## Prerequisites

Before you begin, ensure you have:

- A Xendit account with Terminal Client access
- A valid client key for your environment
- Terminal Gateway service running locally (default port 8189)
- Physical payment terminal device information (Terminal ID and IP address)
- Development environment set up for your chosen platform

<Warning>
Terminal Client requires the Terminal Gateway service to be running locally. Ensure the gateway is properly configured and accessible before integrating the SDK.
</Warning>
