# Terminal Client SDK for iOS (Swift)

The Terminal Client SDK for iOS enables you to integrate secure payment processing directly into your iOS applications. Connect to physical payment terminals through the local gateway service and process transactions seamlessly using our comprehensive Swift SDK.

<Info>
This SDK works alongside the Terminal Gateway service to provide a complete in-person payment solution. You'll use Terminal Client to communicate with the local gateway that manages connections to physical payment devices.
</Info>

<Warning>
**Terminal Gateway Required**: For BRI and NTT terminals, you must install the Terminal Gateway application on your local device before using Terminal Client SDK. See the [Terminal Gateway App Configuration](/api-reference/terminal-gateway/app-configuration) guide for installation and setup instructions.
</Warning>

## Installation

<Warning>
**Prerequisite**: For BRI and NTT terminals, ensure the Terminal Gateway application is installed and running on your local device. See the [Terminal Gateway App Configuration](/api-reference/terminal-gateway/app-configuration) guide for details.

Install the Terminal Client SDK by adding it as a framework in Xcode:

<Step title="Initialize Terminal App">
  Initialize the Terminal App with your client key:
  
  ```swift AppDelegate.swift
  import TerminalCore
  
  func application(
    _ application: UIApplication,
    didFinishLaunchingWithOptions launchOptions: [UIApplication.LaunchOptionsKey: Any]?
  ) -> Bool {
    TerminalApp.shared.initialize(
      clientKey: CLIENT_KEY,
      mode: .production // or .sandbox for testing
    )
    
    return true
  }
  ```
  
  <Warning>
  Replace `CLIENT_KEY` with your actual client key from the Xendit dashboard.
  </Warning>
</Step>

<Step title="Configure Terminal Device">
  Set up your terminal device configuration:
  
  <Tabs>
  <Tab title="Generic Method">

    import TerminalCore
    
    // Configure the default terminal device
    let terminalDevice = TerminalDevice.create(
      id: "TERMINAL_ID",
      ipAddress: "192.168.1.100",
      provider: "BRI" // or "NTT"
    )
    
    TerminalClient.shared.setTerminalDevice(terminalDevice)

  </Tab>
  
  <Tab title="Provider-Specific Method">
    Use provider-specific methods (available without additional dependencies):
    
    ```swift
    import TerminalClient
    import TerminalCore
    
    // For BRI terminals
    let terminalDevice = TerminalDevice.bri(
      id: "TERMINAL_ID",
      ipAddress: "192.168.1.100"
    )
    
    // For NTT terminals
    // let terminalDevice = TerminalDevice.ntt(
    //   id: "TERMINAL_ID",
    //   ipAddress: "192.168.1.100"
    // )
    
    TerminalClient.shared.setTerminalDevice(terminalDevice)
    ```
    
    <Note>
    Provider-specific methods are available in the iOS SDK without requiring additional dependencies. You can use `TerminalDevice.bri()` or `TerminalDevice.ntt()` directly.
    </Note>
  </Tab>
  </Tabs>
  
  <Tip>
  You can also specify a device per request instead of setting a default device.
  </Tip>
</Step>

<Step title="Configure Gateway Port (Optional)">

  ```swift

### Create Payment

Process a payment transaction using async/await:

<Tabs>

  )
  ```
  
  <Note>
  Provider-specific methods are available in the iOS SDK without requiring additional dependencies. You can use `PaymentMethod.card()`, `PaymentMethod.eWallet()`, or `PaymentMethod.qr()` directly.

### Cancel Payment

Cancel an ongoing payment transaction:

```swift
import TerminalClient
import TerminalCore

// Create cancel data
let cancel = Cancel(
  paymentMethod: PaymentMethod.fromValue("card"), // Optional: can be nil
  // or PaymentMethod.card() for provider-specific method
  terminalReference: "123456" // Reference from previous transaction
)

Task {
  do {
    let result = try await TerminalClient.shared.cancelPayment(
      data: cancel,
      device: terminalDevice,
      isSimulation: false
    )
    
    print("Cancellation successful: \(result.status)")
  } catch let error as TerminalException {
    print("Cancellation failed: \(error.message)")
  }
}
```

<Tip>
You can create a `Cancel` object from a previous `TerminalResult` using `Cancel.fromResult(result)`.
</Tip>

### Get Transaction History

Retrieve transaction history from the terminal:

```swift
import TerminalClient

// Create histories data with optional filters
let histories = Histories(
  commands: ["PAY", "CANCEL"], // Optional: filter by command types
  statuses: [.success], // Optional: filter by statuses
  from: "2024-01-01T00:00:00Z", // Optional: ISO 8601 format start date
  to: "2024-01-31T23:59:59Z" // Optional: ISO 8601 format end date
)

Task {
  do {
    let history = try await TerminalClient.shared.getHistory(
      data: histories,
      device: terminalDevice
    )
    
    print("Retrieved \(history.count) transactions")
    for transaction in history {
      print("Transaction: \(transaction.status)")
    }
  } catch let error as TerminalException {
    print("Failed to retrieve history: \(error.message)")
  }
}
```

## Error Handling

### TerminalException

The SDK throws `TerminalException` for terminal-specific errors:

```swift
import TerminalClient
import TerminalCore

let payment = Payment(
  orderID: "ORDER_123",
  amount: 10000.0,
  paymentMethod: PaymentMethod.fromValue("card")
  // or PaymentMethod.card() for provider-specific method
)

do {
  let result = try await TerminalClient.shared.createPayment(payment)
} catch let error as TerminalException {
  switch error.error.code {
  case .failedToConnect:
    // Terminal Gateway service not reachable
    print("Cannot connect to Terminal Gateway")
  case .authenticationFailed:
    // Invalid client key or terminal credentials
    print("Authentication failed")
  case .terminalBusy:
    // Terminal is processing another transaction
    print("Terminal is busy")
  default:
    print("Error: \(error.error.message)")
  }
} catch {
  print("Unexpected error: \(error.localizedDescription)")
}
```

### Error Structure

```swift
struct TerminalException: Error {
  let httpStatus: Int
  let error: ApiError
}

struct ApiError {
  let code: ErrorCode
  let message: String
}
```

## Troubleshooting

### Common Issues and Solutions

<AccordionGroup>
<Accordion title="Cannot Connect to Terminal Gateway">
  **Problem**: SDK cannot reach the Terminal Gateway service.
  
  **Solutions**:
  1. Verify Terminal Gateway service is running locally
  2. Check the port configuration (default: 8189)
  3. Ensure the terminal device IP address is correct
  4. Verify network connectivity between your app and gateway
  
  ```swift
  // Verify port configuration
  TerminalClient.shared.setPort("8189")
  
  // Check terminal device IP (using generic method)
  let device = TerminalDevice.create(
    id: "TERMINAL_ID",
    ipAddress: "192.168.1.100", // Verify this IP
    provider: "BRI"
  )
  
  // Or using provider-specific method
  // let device = TerminalDevice.bri(
  //   id: "TERMINAL_ID",
  //   ipAddress: "192.168.1.100"
  // )
  ```
</Accordion>

<Accordion title="No Terminal Device Provided">
  **Problem**: Error indicating no terminal device is configured.
  
  **Solution**: Set a default terminal device before making requests:
  
  ```swift
  // Set default device
  TerminalClient.shared.setTerminalDevice(terminalDevice)
  
  // Or provide device per request
  let payment = Payment(
    orderID: "ORDER_123",
    amount: 10000.0,
    paymentMethod: PaymentMethod.fromValue("card")
    // or PaymentMethod.card() for provider-specific method
  )
  
  Task {
    let result = try await TerminalClient.shared.createPayment(
      data: payment,
      device: terminalDevice
    )
  }
  ```
</Accordion>

<Accordion title="Authentication Failed">
  **Problem**: Requests fail with authentication errors.
  
  **Solutions**:
  1. Verify TerminalApp is initialized with correct client key
  2. Check terminal ID matches the configured device
  3. Ensure client key has Terminal Client permissions
  
  ```swift
  // Re-initialize with correct client key
  TerminalApp.shared.initialize(
    clientKey: CLIENT_KEY,
    mode: .production
  )
  ```
</Accordion>

<Accordion title="Network Errors">
  **Problem**: Network-level errors when communicating with gateway.
  
  **Solutions**:
  1. Check network connectivity
  2. Verify firewall settings allow connections to gateway port
  3. Ensure Terminal Gateway service is accessible
  4. Check terminal device IP address is reachable
  
  <Tip>
  Use network debugging tools to verify connectivity between your app and the Terminal Gateway service.
  </Tip>
</Accordion>

<Accordion title="Async/Await Compatibility">
  **Problem**: Issues with async/await syntax or compatibility.
  
  **Solutions**:
  1. Ensure you're using iOS 15+ or macOS 12+ for native async/await support
  2. Use callback-based methods if targeting older iOS versions
  3. Wrap async calls in Task blocks
  
  ```swift
  import TerminalClient
  import TerminalCore
  
  let payment = Payment(
    orderID: "ORDER_123",
    amount: 10000.0,
    paymentMethod: PaymentMethod.fromValue("card")
    // or PaymentMethod.card() for provider-specific method
  )
  
  // For iOS 15+
  Task {
    let result = try await TerminalClient.shared.createPayment(data: payment)
  }
  
  // For older iOS versions, use callbacks
  TerminalClient.shared.createPaymentAsync(data: payment) { result in
    // Handle result
  }
  ```
</Accordion>
</AccordionGroup>

## Best Practices

### Error Handling

Always implement comprehensive error handling:

```swift
func processPaymentSafely(orderID: String, amount: Double) async throws -> TerminalResult {
let payment = Payment(
  orderID: orderID,
  amount: amount,
  paymentMethod: PaymentMethod.fromValue("card")
  // or PaymentMethod.card() for provider-specific method
)
  
  do {
    return try await TerminalClient.shared.createPayment(payment)
  } catch let error as TerminalException {
    switch error.error.code {
    case .terminalBusy:
      // Retry after delay
      try await Task.sleep(nanoseconds: 5_000_000_000) // 5 seconds
      return try await processPaymentSafely(orderID: orderID, amount: amount)
    case .failedToConnect:
      // Check connectivity and retry
      throw error
    default:
      throw error
    }
  }
}
```

### Task Management

Properly manage async tasks:

```swift
class PaymentViewController: UIViewController {
  private var paymentTask: Task<Void, Never>?
  
  func processPayment() {
    let payment = Payment(
      orderID: "ORDER_123",
      amount: 10000.0,
      paymentMethod: PaymentMethod.fromValue("card")
      // or PaymentMethod.card() for provider-specific method
    )
    
    paymentTask = Task {
      do {
        let result = try await TerminalClient.shared.createPayment(data: payment)
        await MainActor.run {
          // Update UI on main thread
          self.updateUI(with: result)
        }
      } catch {
        await MainActor.run {
          // Handle error on main thread
          self.showError(error)
        }
      }
    }
  }
  
  deinit {
    paymentTask?.cancel()
  }
}
```
