## Choose Your Integration Mode

Compare Host-to-Host (H2H) and Client-to-Client (C2C) modes to select the best approach for your needs.

| Feature | **Host-to-Host (H2H)** ✅ **RECOMMENDED** | Client-to-Client (C2C) |
|---------|-------------------------------------------|-------------------------|
| **Deployment Model** | Centralized fleet management through Gateway App | Direct terminal-to-app communication over local network |
| **Device Management** | Managed by Xendit Terminal Gateway | Self-managed by your application |
| **API Model** | Managed payment sessions with callbacks | Direct device commands (pay, cancel, receipt, settlement) |
| **SDK** | H2H SDK (Android/iOS) for device connectivity | C2C SDK (Android/iOS) available as optional convenience wrapper |
| **Network Requirements** | Internet connectivity required | Local network or internet connectivity |
| **Offline Support** | Limited (requires connection to Gateway) | Better offline capabilities |
| **BRI/NTT Terminals** | Require Gateway App or H2H SDK | Require Gateway App or C2C SDK |
| **Customization** | Standard workflows | Full control over payment flows |
| **Time to Market** | Faster — standard implementation | Longer — custom development required |
| **Best For** | Standard payment acceptance, quick deployment | Custom POS systems, kiosks, specialized workflows |
