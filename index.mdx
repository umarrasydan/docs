## Welcome to Xendit Terminal

Xendit Terminal provides comprehensive in-person payment solutions for businesses using physical terminal devices. Process secure payments through Terminal API and manage terminal devices using Terminal Gateway across Android, iOS, and desktop platforms.

<Info>
**New to Xendit Terminal?** Start with our [quickstart guide](/guides/getting-started/quickstart) to get your first terminal payment processed in under 10 minutes.
</Info>

## Prerequisites

Before you begin, ensure you have:

- Terminal API access and API key from the Xendit In-Person Payment team
- Terminal Gateway set up (desktop app or Android/iOS SDK)
- A terminal with Terminal ID and network connectivity
- Your environment base URL:
  - Development: `https://terminal-dev.xendit.co`
  - Production: `https://terminal.xendit.co`

<Warning>
Terminal API uses HTTP Basic Auth with your Terminal API key as the username and an empty password. Always include a trailing colon when constructing the header.
</Warning>

## How it works

1. **Create a session** with Terminal API for an amount, currency, and terminal.
2. **Execute on device** through Terminal Gateway; the customer taps/inserts/swipes.
3. **Receive callbacks** for real-time status updates.
4. **Query details** for reconciliation or show receipts.

## Integration Paths

<Tabs>
<Tab title="Retail POS System">
  <Steps>
  <Step title="Obtain Terminal API key">
    Contact the Xendit In-Person Payment team to request your Terminal API key for authentication.
    
    <Check>
    You can successfully authenticate to the development base URL with your key.
    </Check>
  </Step>
  <Step title="Set up Terminal Gateway">
    Install Terminal Gateway app or integrate mobile SDK on your devices to manage physical terminals.
    
    <Check>
    The device appears connected and ready in your Gateway.
    </Check>
  </Step>
  <Step title="Register terminal devices">
    Configure your physical terminals with Terminal ID and IP address in the Gateway application.
  </Step>
  <Step title="Create payment sessions">
    Use Terminal API to create payment sessions and process transactions on connected terminals.
    
    <Check>
    You receive a session ID and the terminal prompts for payment.
    </Check>
  </Step>
  <Step title="Handle callbacks">
    Set up webhook endpoints to receive real-time payment status updates and notifications.
    
    <Tip>
    Use unique `Idempotency-key` values per create request to safely retry.
    </Tip>
  </Step>
  </Steps>
</Tab>

<Tab title="Mobile POS App">
  <Steps>
  <Step title="Install Terminal Gateway SDK">
    Add Terminal Gateway SDK to your Android or iOS application for terminal device management.
  </Step>
  <Step title="Initialize SDK">
    Configure the SDK with your client key and add terminal providers (BRI, GHL, etc.).
  </Step>
  <Step title="Register devices">
    Register your physical terminal devices with Terminal ID and connection details.
  </Step>
  <Step title="Process payments">
    Use Terminal API to create payment sessions that automatically trigger on registered terminals.
  </Step>
  <Step title="Monitor transactions">
    Observe connection states and handle payment results through SDK callbacks.
  </Step>
  </Steps>
</Tab>
</Tabs>
